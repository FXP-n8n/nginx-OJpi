# 4-Eyes Dashboard — CLAUDE.md

## What this directory is

`dashboard/` is the source for `dashboard.4-eyes.eu`, one of three sites served by the Railway nginx container `nginx-4Eyes`.

## Pages

- `index.html` — fleet health dashboard (cron jobs, agent freshness)
- `sciencebusiness.html` — ScienceBusiness tracker page

## Design language

Follow the shared 4-Eyes outbound design language (`../design.md`):

| Token | Hex | Usage |
|---|---|---|
| Deep Ink | `#1B2D4F` | Page background, card backgrounds |
| Signal Amber | `#E8A020` | Headings, links, dashboard accents |
| Slate | `#4A5568` | Secondary text |
| Mist | `#F4F6F9` | Light accents |

- **Typography:** Inter 400/500/600/700; JetBrains Mono for data and numbers.
- **Layout:** Simple dark dashboard, single-column cards on mobile.

## How to change things

Styles are mostly inline in each HTML file for simplicity. Update the CSS in the `<style>` block at the top of `index.html` or `sciencebusiness.html`.

## Data freshness

The fleet dashboard is regenerated by cron jobs on the Hermes Agent Railway service. The file in this repo is a snapshot. If you want the live dashboard to reflect repo changes, the cron job that writes it must be updated separately (it lives in the Hermes agent project, not here).

## Deployment

This site is deployed as part of the multi-site repo:

```bash
cd ..
git add .
git commit -m "refactor(dashboard): update dashboard site"
git push origin main
# Railway auto-deploys
```

`nginx.conf` also copies `sciencebusiness.html` to the chatbot document root so the URL `dashboard.4-eyes.eu/sciencebusiness.html` works.

## Common mistakes

- Don't lighten the dashboard background too much — it should stay dark (Deep Ink).
- Don't introduce new colors outside the design tokens.
- Don't expect live data when testing locally; the dashboard HTML is a static snapshot.
